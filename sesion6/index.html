Descripción de la práctica

EEsta práctica consiste en el desarrollo de varias aplicaciones web utilizando la API WebRTC, que permite integrar comunicación en tiempo real directamente en el navegador sin necesidad de software adicional. En primer lugar, se captura el flujo de vídeo de la cámara web mediante getUserMedia y se muestra en un elemento video. A continuación, se amplía la aplicación para permitir la descarga como imagen PNG utilizando un elemento canvas. Después, se implementa una llamada de vídeo local entre dos extremos usando RTCPeerConnection. Finalmente, se integra WebRTC con WebGL adaptando una escena Three.js de una sesión anterior para utilizar el flujo de vídeo de la cámara como textura dinámica.

Ejercicio 1

En el primer ejercicio se que accede a la web del dispositivo y muestra el flujo de vídeo capturado en un elemento video con reproducción automática. Para ello se define un constraints indicando que solo se necesita vídeo (sin audio), y el flujo obtenido se asigna directamente a la propiedad srcObject del elemento. No se encontraron dificultades durante la implementación.

Ejercicio 2

En el segundo ejercicio se amplía la página anterior. Se incorpora un elemento canvas oculto y un enlace con el atributo download que, al pulsarse, copia el fotograma actual del vídeo sobre el canvas mediante drawImage y genera una URL de descarga en formato PNG usando toDataURL. No se encontraron dificultades durante la implementación.

Ejercicio 3

En el tercer ejercicio se implementa una llamada de vídeo local entre dos extremos usando RTCPeerConnection. Se configuran los parámetros de conexión y se gestionan los eventos necesarios para establecer la comunicación. No se encontraron dificultades durante la implementación.

Ejercicio 4

En el cuarto ejercicio se integra WebRTC con WebGL adaptando una escena Three.js de una sesión anterior para utilizar el flujo de vídeo de la cámara como textura dinámica. Se crea una textura de vídeo a partir del elemento video y se actualiza en cada frame para reflejar el contenido en tiempo real. No se encontraron dificultades durante la implementación.
